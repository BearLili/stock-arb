# M2 设计：纸面撮合 + PnL 归因 + MEXC 滞后校正

配套 PRD §4/P1、开发文档 §4.3。数据源 = 服务器回流的 tick 级 BBO JSONL（`data/live/*/*.jsonl.gz`）。

## 1. 统一策略接口（backtest/paper/live 同签名，开发文档 §2）

```ts
interface Strategy {
  onBbo(e: BboEvent): Intent[];
  onFunding(e: FundingEvent): Intent[];
  onFill(f: Fill): Intent[];
}
```
`Intent` = 想在某 (sym,prod) 以 taker/maker 下单某方向某数量；撮合器把 Intent 变成 Fill。同一策略对象在三模式下复用，仅数据源与撮合器不同。

## 2. 回放（replay.ts）
按 `ts_recv` 全局有序合并多产品 JSONL，逐条喂 `BboEvent`。**MEXC 腿特殊处理**：见 §5。

## 3. 纸面撮合（matcher.ts）—— 开发文档 §4.3

- **taker**：以对侧 bbo 价立即成交 + **1bp 滑点保守项**（买加价、卖减价 1bp）。
- **maker**：在同侧 bbo 挂单，成交判定 = **对侧价穿越挂价**（用 tick 数据回放：挂 bid=P，后续对手 ask≤P 即成交；挂 ask=P，后续 bid≥P 即成交）。挂单未成交超时撤单。
- **maker 成交率**：挂单数 / 成交数，**M2 最重要输出**。S2 依赖 Gate maker −1bp 返佣，**若真实成交率<30% 需重估 S2**（报告显式给结论）。

## 4. PnL 四项归因（report.ts，FR10）
每笔/每日拆：**价差**（进出中间价差）｜**资金费**（持仓期 net carry 累计）｜**手续费**（taker/maker 实际费，含 Gate 返佣）｜**滑点**（taker 1bp + maker 成交价 vs 期望差）。

## 5. 滞后校正的权威重放语义（用户裁决，所有腿统一，非 MEXC 特判）

**原则：信号看到什么就用什么（陈价）；成交按订单真实到达时刻的报价撮合。**

1) **信号形成**：只用 `ts_recv` 对齐的数据流（实盘引擎真实看到的、含陈价的视图）。
   **禁止用 `ts_exch` 对齐**——那是实盘看不到的上帝视角。策略在决策时刻 t（触发事件的 ts_recv）产生 Intent。
2) **成交模拟**：订单于 `t + rtt_venue` 到达交易所。
   - **taker 成交价** = 录制流中该 (sym,prod) **第一条满足 `ts_exch ≥ t + rtt_venue`** 的报价的对侧价 + 滑点项（1bp）。即按交易所事件时间向前走，跳过所有在订单到达前已过期的陈价。
   - **maker**：挂 bbo 同侧价 P（决策时看到的，可能陈）；订单自 `t+rtt` 起挂在 P，成交判定 = 后续报价（`ts_exch ≥ t+rtt`）对侧价穿越 P（挂 bid→ask≤P；挂 ask→bid≥P）；超时未穿越则撤单。成交价 = P。
   - **滞后**用每笔报价自测值 `ts_recv − ts_exch`（可变 0.3–3.2s），**不用常数**。
3) **rtt 参数**：进 config `rtt_ms`（占位 MEXC 300 / BN·Gate 50ms），服务器部署后实测替换，**不许硬编码**。
4) **两套口径并行**：
   - **未校正（naive）**：成交按决策时刻 t 看到的（可能陈）报价立即成交（rtt=0、用决策时观测价）。= 监控端乐观口径。
   - **校正后（lag-aware）**：如 (2)，按到达时刻真实报价成交。
   两者差值 = **陈价幻象的定量**。
5) **服务器复测**：mexcperp（及各腿）滞后分布须在服务器实测（东京可能显著改善），滞后分布是 **config 数据**不是代码常数，本地实测仅占位。

**产出与门槛**：每策略「未校正 vs 校正后」纸面 PnL + 机会捕获率对比表（逐笔/逐日/汇总）。
**S1/S2 进 M3 的裁决门槛：校正后连续 ≥5 个交易日纸面 PnL 为正。**

## 5b. tradeable 双保险
config 每产品加 `tradeable` 标记；`mexcon: false`（PRD §3.4 仅观察，点差 100bp 级）。撮合层直接**拒绝生成 tradeable=false 腿的 Intent**，防未来策略配置误用。

## 6. 交付物
- `scripts/paperRun.ts`：跑纸面撮合 → PnL 报表 + maker 成交率 + 滞后校正对比。
- 连续 5 个交易日纸面 PnL 报表（P1 验收：正收益且回撤可解释）。
- 数据不足（服务器 JSONL 未回流）时如实标注，不出伪结论。
