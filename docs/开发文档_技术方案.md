# 股票代币跨所套利系统 — 开发文档（技术方案）

版本 v1.0 ｜ 2026-07-22 ｜ 配套《需求文档_股票代币套利系统PRD.md》使用

---

## 1. 技术选型建议

- 语言：**Node.js 20+ / TypeScript**（业主指定，且本系统以WS长连接与事件处理为主，Node原生契合；信号是分钟级均值回归而非抢单型，无需更低延迟语言）
- 依赖（保持精简）：`ws`（WebSocket客户端）、`undici`（REST）、`better-sqlite3`（运行状态/订单对账）、`zod`（配置校验）、`pino`（结构化日志）。落盘用按日分区的 JSONL（gzip），分析查询用 `duckdb`（node包）直查JSONL/导出Parquet——避免引入重型消息队列
- 原型关系：`ws_monitor.py` 是**接入规格参考**（各所订阅报文格式、ping要求、重连、边界情况全在里面），M0 用 TS 依此重写，不复用其代码本身；`monitor_config.json` 原样复用
- 历史回测校准：`gap_data/` 内 JSON/CSV 直接可被 TS 读取；如需重算统计，允许保留 Python 分析脚本作为 offline 工具（不进主工程runtime），或用 duckdb SQL 重写
- 部署：单服务器（东京/新加坡机房，BN/Gate/Bybit撮合主要在亚洲，HL节点东京），三进程：`collector`（行情+落盘）、`engine`（信号+告警+纸面）、`executor`（下单，独立密钥，P2起）。进程间本机 unix socket / 轻量 IPC
- 密钥：环境变量注入，交易密钥禁提现权限，executor 与外网告警通道物理隔离

## 2. 架构

```
[9路WS/REST接入] → [Normalizer 统一符号/时钟] → [BBO状态机]
                                        ↓                ↓
                               [Parquet落盘]      [净Edge引擎 + 基线]
                                                        ↓
                                    [资金费引擎(REST轮询)] → [信号生成]
                                                        ↓
                            [告警(TG/CSV)] ← [策略路由] → [纸面撮合器 | Executor]
                                                        ↓
                                                  [风控闸门 + 审计日志]
```

核心接口（backtest/paper/live 三模式同一签名）：
```typescript
interface Strategy {
  onBbo(e: { sym: string; prod: string; bid: number; ask: number; ts: number }): Intent[];
  onFunding(e: { sym: string; prod: string; rate: number; nextTs: number }): Intent[];
  onFill(order: Fill): Intent[];
}
```

### 3.3 符号命名陷阱（务必用映射表，勿字符串拼接）
- Gate 永续混合命名：TSLAX_USDT/NVDAX_USDT/CRCLX_USDT 带X，但 **MU_USDT/SNDK_USDT 不带X**；且 Gate 有 MUU_USDT（≠美光）、SPYM/SPYX 等易混对
- MEXC：现货 Ondo 为 {SYM}ONUSDT，永续为 {SYM}STOCK_USDT
- OKX：X前缀 X{SYM}-USDT；Binance bStocks：{SYM}BUSDT；已验证价格交叉校验法：新增符号先比对现价与 bStocks 同标的（±2%内）防止认错合约

## 4. 核心算法

### 4.1 可执行净edge（唯一真实口径，告警用）
```
edge_sellA_buyB = (bidA − askB) / mid × 1e4 − takerA − takerB    # bp
```
基线：EWMA(α≈0.001/tick) 与 240min 滚动中位数双轨，取更保守者；偏离 dev = mid_diff − baseline。
去噪：单tick尖刺过滤（连续3个tick确认）；陈旧腿丢弃（>10s）；HL周末休市时暂停含HL的对。

### 4.2 资金费net carry（持仓对）
pair_carry_bp_per_day = (funding_short_leg − funding_long_leg) × 10000 × 3（8h所）
HL为每小时结算×24。S2类持仓对每日结算时点前重估是否继续持有。

### 4.3 纸面撮合假设
taker：对侧bbo价立即成交，加1bp滑点保守项；maker：挂bbo同侧，成交判定=对侧价穿越挂价（用tick数据回放），成交率照实统计——**maker策略的纸面成交率是P1最重要输出**（S2依赖maker返佣，若真实成交率<30%需重估）。

## 5. 执行模块（P2）

- 两腿顺序：先执行流动性差/更难成交的腿（MEXC/HL侧），成交确认后立即taker对冲BN侧（最深）；对冲腿超时1s未成交→市价强平先成交腿并告警
- 部分成交：按已成交数量对冲，剩余撤单
- 幂等：clientOrderId 全局唯一，重启后对账（REST查持仓+挂单恢复状态机）
- kill switch：一键全撤单+全平仓+降级为监控模式（必须最先实现并演练）

## 6. 风控参数（初始值，可配置）

单笔名义 ≤ $500（P2）→ $5,000（P3）；单策略净敞口 ≤ $20,000；总敞口 ≤ $50,000；日亏损熔断 = 日均预期盈利×3；API连续错误>5次/分钟熔断该所；HL对周五19:00 ET前强制平仓（休市敞口规则）；财报日（SNDK/MU日历）事件前2h暂停开新仓。
